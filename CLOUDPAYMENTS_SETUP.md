# Настройка CloudPayments для TryVice

Для того чтобы оплата тарифов автоматически активировала лимиты в приложении, необходимо выполнить следующие настройки в личном кабинете CloudPayments.

## 1. Настройка уведомлений (Webhooks)

Перейдите в раздел **Настройки** -> **Уведомления** в личном кабинете CloudPayments.

### Pay уведомление (Обязательно)
Это уведомление отправляется при успешной оплате.

*   **Адрес (URL):** `https://your-domain.ru/api/webhook/cloudpayments` (замените `your-domain.ru` на ваш актуальный домен).
*   **Метод:** `POST`
*   **Формат:** `JSON`

### Check уведомление (Опционально)
Используется для проверки возможности платежа перед его совершением.
*   Если вы хотите проверять существование магазина перед оплатой, укажите тот же URL.
*   Наш текущий эндпоинт вернет `{ code: 0 }`, что разрешит платеж.

## 2. Секретные ключи

В файле `.env.local` (или в настройках хостинга Timeweb) должны быть установлены следующие переменные:

```env
# Public ID из личного кабинета CP
NEXT_PUBLIC_CLOUDPAYMENTS_PUBLIC_ID=pk_...

# API Secret (Пароль для API) из раздела Настройки -> API
CLOUDPAYMENTS_API_SECRET=...
```

## 3. Формат данных при запуске виджета

При вызове виджета CloudPayments на фронтенде (в компоненте оплаты), необходимо передавать объект `data` с ID магазина и тарифа. 

Пример кода на фронтенде:

```javascript
const widget = new cp.CloudPayments();
widget.pay('auth', {
    publicId: 'pk_...', // ваш publicId
    description: 'Оплата тарифа Базовый',
    amount: 2900,
    currency: 'RUB',
    accountId: user.email, // опционально
    data: {
        shop_id: 'uuid-вашего-магазина',
        tariff_id: 'uuid-тарифа',
        type: 'tariff'
    }
}, {
    onSuccess: (options) => { /* действие при успехе */ },
    onFail: (reason, options) => { /* действие при ошибке */ },
});
```

## 4. Безопасность

Эндпоинт проверяет подпись `X-Content-Hmac`. Убедитесь, что в настройках уведомлений CloudPayments включена отправка HMAC-подписи и секретный ключ совпадает с `CLOUDPAYMENTS_API_SECRET`.
