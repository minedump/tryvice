# TryVice Widget Expert

Этот скилл содержит контекст и правила для разработки и поддержки клиентского виджета TryVice.

## Архитектура

Виджет реализован как Native Web Component (`tryvice-widget`) с использованием Shadow DOM для изоляции стилей. Код разбит на модули в директории `src/widget/` и собирается в `public/widget.js` с помощью `esbuild`.

### Структура файлов
- `src/widget/index.ts`: Основной класс, управление состоянием (state), обработка событий и запросы к API.
- `src/widget/styles.ts`: CSS-стили виджета. Используются CSS-переменные для кастомизации (например, `--primary`).
- `src/widget/templates.ts`: HTML-шаблоны шагов и встроенные SVG-иконки (спрайты).
- `public/widget.js`: Скомпилированный и минифицированный бандл для клиентов.

## Правила разработки

1. **Изоляция**: Все стили должны быть внутри Shadow DOM. Не использовать внешние CSS-библиотеки.
2. **SVG**: Иконки должны быть встроены в `templates.ts` через `<symbol>` и использоваться через `<use href="#id">`. Это предотвращает проблемы с CORS.
3. **Сборка**: После любых изменений в `src/widget/` необходимо запускать `npm run build:widget`.
4. **API**: Виджет общается с `/api/widget/*`. Базовый URL определяется автоматически из `document.currentScript.src`.
5. **Состояние**: Состояние виджета хранится в `this.state`. При изменении состояния вызывается `this.render()`.

## Команды
- `npm run build:widget`: Сборка и минификация.
- `npm run watch:widget`: Режим отслеживания изменений.

## Визуальные стандарты
- Скругления: 48px для модального окна, 24px для карточек.
- Шрифты: Inter, системные санс-серифы.
- Цвета: Основной цвет берется из настроек магазина (`primary_color`).
