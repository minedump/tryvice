# Навык: Эксперт по проекту TryVice (ТрайВайс)

**Статус проекта:** Основная разработка завершена. Текущий этап: Исправление ошибок (Bugfixing) и стабилизация.

Этот навык содержит контекст, правила разработки и архитектурные решения для проекта TryVice.

## Обзор проекта
TryVice — это SaaS для виртуальной примерки одежды. 
- **Виджет:** Встраиваемый JS-скрипт (`public/widget.js`).
- **Админка:** Next.js приложение в `src/app`.
- **Бэкенд:** Supabase (RLS включен) + Next.js API Routes.

## Архитектурные правила
1. **База данных:** Используй миграции в `supabase/migrations/`. Всегда учитывай RLS (Row Level Security).
2. **API:** Все эндпоинты виджета находятся в `src/app/api/widget/`, админки — в `src/app/api/admin/`.
3. **AI:** Логика работы с NanoBanana API инкапсулирована в `src/lib/ai-service.ts`.
4. **Стили:** Tailwind CSS. Компоненты UI лежат в `src/components/`.
5. **Контекст:** Для управления состоянием магазина на фронтенде используй `src/context/ShopContext.tsx`.
6. **Окружение:** В терминале (PowerShell) оператор `&&` не работает. Выполняй команды последовательно.

## Ключевые эндпоинты (API)
- `POST /api/widget/init` — Проверка shop_id и домена.
- `POST /api/widget/tryon` — Запуск генерации.
- `POST /api/admin/products/sync` — Парсинг XML-фида.

## Правила работы с изображениями
- Пользовательские фото и результаты хранятся в Supabase Storage.
- Срок хранения: 180 дней.
- Перед генерацией обязательна проверка через `ai-service.ts` (наличие человека, качество фона).
